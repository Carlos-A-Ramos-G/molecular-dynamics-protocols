# FEP Serial — AMBER Thermodynamic Integration workflow

A Python workflow manager for alchemical free-energy perturbation (FEP) calculations
using AMBER thermodynamic integration (TI) with Gauss-Legendre quadrature.
Supports three execution modes: **local** (single-GPU workstation), **serial** (SLURM,
one job at a time), and **parallel** (SLURM, all replicas simultaneously).

## Example system: LCN → ACN

```
================================================================================
  FEP THERMODYNAMIC INTEGRATION: FCE-LTL-GQU-LCN  -->  FCE-LTL-GQU-ACN
================================================================================

      F3C                                     F3C
        \                                      \
         C=O                                    C=O
         |                                      |
        HN                                     HN
         |                                      |
     tBu-CH                                 tBu-CH
          \                                     \
           C=O                                   C=O
           |                                     |
           N--[bicycle]         FEP/TI           N--[bicycle]
           |                ──────────────►      |
           C=O               (DeltaDeltaG)       C=O
           |                                     |
          NH                                    NH
           |                                     |
          CH--CH2                               CH--CH3   <-- MUTATION
           |       \                             |
          C=N    O=C--CH2                       C=N
                 |       |
                 NH------/
              (lactam ring)

  FCE-LTL-GQU-LCN                        FCE-LTL-GQU-ACN

--------------------------------------------------------------------------------
  MUTATION SITE:

  LCN:  ...NH-CH(CN)-CH2-C(=O)-NH-CH2...   <- gamma lactam ring
                          |______________|
                            lactam ring

  ACN:  ...NH-CH(CN)-CH3                   <- methyl

--------------------------------------------------------------------------------
  UNCHANGED SCAFFOLD:
  F3C-C(=O)-NH -- tBu-CH -- C(=O)-N-[GQU bicycle] -- C(=O)-NH -- CH(CN)


```

## Prerequisites

| Requirement | Version |
|---|---|
| Python | ≥ 3.8 |
| numpy | any recent |
| pyyaml | any recent |
| AMBER | ≥ 20 (pmemd.cuda + cpptraj) |
| SLURM | only for `serial` / `parallel` modes |

Install Python dependencies:

```bash
pip install -r requirements.txt
```

---

## Quick start

```bash
# 1. Edit config.yaml for your mutation
#    (resnew/resold, masks, topology file names, simulation parameters)

# 2. Generate all input files
python fep_runner.py setup --mode local

# 3. (optional) edit the generated scripts for cluster-specific requirements

# 4. Submit without regenerating files
python fep_runner.py submit --mode local

# 5. After completion, compute ΔΔG
python fep_runner.py analyse
```

## Configuring a new mutation

Edit **`config.yaml`** — it is the only file you need to touch between mutations.

```yaml
resnew: LCN           # three-letter code of the appearing residue
resold: ACN           # three-letter code of the disappearing residue
                      # system_name is derived automatically as resnew_to_resold

n_lambdas: 9          # Gauss-Legendre lambda windows (>= 3; common: 5, 9, 12)
replicates: 1         # independent replicas

amber:
  cuda_module: amber/24           # SLURM module for GPU runs
  cpu_module:  amber/24           # SLURM module for CPU equilibration
  cuda_lib_path: /path/to/nvidia-driver-libs  # leave empty if not needed

slurm:
  gpu:
    time: "24:00:00"
    ntasks: 1
    gres: gpu:1
    partition: partition_name
    # Any key here is written verbatim as #SBATCH --key=value.
    # Add qos, account, mem, etc. without editing fep_runner.py.
  cpu:
    ntasks: 4         # substituted into execution_command.cpu below

execution_command:
  gpu: "srun $AMBERHOME/bin/pmemd.cuda"
  # {ntasks} is replaced automatically with slurm.cpu.ntasks.
  cpu: "srun -n {ntasks} $AMBERHOME/bin/pmemd.MPI"

simulation:
  min:
    maxcyc: 10000
  equil:
    nstlim: 200000
    dt: 0.001
    ntwx: 1000
  prod:
    nstlim: 5000000
    dt: 0.001
    ntwe: 1000
    ntwx: 10000

systems:
  unbounded:
    parameters:  ligands_solvated.parm7
    coordinates: ligands_solvated.rst7
    timask1: "':1-4'"
    timask2: "':5-8'"
    scmask1: "'@54-65'"
    scmask2: "'@121'"

  bounded:
    parameters:  complex_solvated.parm7
    coordinates: complex_solvated.rst7
    timask1: "':608-611'"
    timask2: "':612-615'"
    scmask1: "'@9344-9355'"
    scmask2: "'@9411'"
```

Place the topology (`.parm7`) and coordinate (`.rst7`) files for both the
unbounded and bounded systems inside `setup_files/<resnew>_to_<resold>/`.

## Execution modes

### local — single-GPU workstation

```bash
# Generate input files
python fep_runner.py setup --mode local

# (optional) edit run_local.sh scripts

# Launch (both legs chain sequentially in background)
python fep_runner.py submit --mode local

# Monitor
tail -f LCN_to_ACN/unbounded/run.log
tail -f LCN_to_ACN/bounded/run.log
```

The legs run one after the other (unbounded first, then bounded) so the GPU
is never double-booked. Each `run_local.sh` runs minimisation → equilibration
→ all production windows in the order: mid → mid-1 → … → 1 → mid+1 → … → N.

### serial — SLURM, one job at a time

```bash
# Generate scripts
python fep_runner.py setup --mode serial

# (optional) edit FEP_MIN.cmd / FEP_EQUIL.cmd / FEP_PROD_*.cmd

# Submit
python fep_runner.py submit --mode serial
```

Jobs chain via `sbatch`: equil → window-mid → mid-1 → … → 1 →
mid+1 → … → N. Replicas are chained end-to-end (replica 2 starts when
replica 1's last window finishes).

### parallel — SLURM, all replicas simultaneously

```bash
python fep_runner.py setup --mode parallel
python fep_runner.py submit --mode parallel
```

After equilibration, every replica's central window is submitted at once.
Within each replica the windows fan out bidirectionally:
```
1 ← 2 ← 3 ← 4 ← 5 → 6 → 7 → 8 → 9
```

## Analysis

```bash
python fep_runner.py analyse            # uses last 4000 dV/dλ records
python fep_runner.py analyse --tail 2000
bash analyse.sh --tail 2000             # identical — thin wrapper
```

Reads all `ti{replica}_{n}.en` energy files, extracts dV/dλ (L9 records), computes
per-replica ΔG via Gauss-Legendre quadrature, and prints:

```
ΔΔG = ΔG(bounded) − ΔG(unbounded)
ΔΔG = +X.XXX ± X.XXX kcal/mol
```

## Testing

```bash
python -m pytest test_fep_runner.py -v
```

The suite runs `setup --mode local` in a temporary directory and validates:

- GL lambda values for all 9 windows
- All four alchemical masks (`timask1/2`, `scmask1/2`) against expected values
- Simulation parameters (`nstlim`, `dt`, `ntwe`, `ntwx`)
- `run_local.sh` generated and executable for both legs
- Unbounded masks not contaminated by bounded values

## Utilities

### clean.sh

Remove all files generated by `fep_runner.py setup` and any AMBER simulation
outputs, reading the system name and legs from `config.yaml`:

```bash
bash clean.sh            # interactive confirmation
bash clean.sh --dry-run  # preview only
bash clean.sh --yes      # skip confirmation
```

### fep_runner.py — full CLI reference

```
python fep_runner.py [--config FILE] setup   [--mode {local,serial,parallel}] [--submit]
python fep_runner.py [--config FILE] submit  [--mode {local,serial,parallel}]
python fep_runner.py [--config FILE] analyse [--tail N]
```

| Command | Description |
|---|---|
| `setup` | Generate all input files and SLURM/local job scripts. Pass `--submit` to also launch immediately after generation. |
| `submit` | Submit existing job scripts without regenerating any files. Use this after manually editing the generated scripts for cluster-specific requirements. `--mode` must match the mode used during setup. |
| `analyse` | Read all `.en` energy files, extract dV/dλ, and compute ΔΔG. |
